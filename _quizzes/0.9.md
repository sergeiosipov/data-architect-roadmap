# Phase 0 · 0.9 pandas & JupyterLab — 100 self-test questions

Q: What is JupyterLab?
A: A browser-based interactive development environment for notebooks, code, and data, where you run Python in cells and see output (text, tables, charts) inline beneath each cell.

Q: What is a Jupyter notebook cell?
A: An individually executable unit of a notebook; a code cell runs Python and shows its output below, while a Markdown cell renders formatted text.

Q: How do you run the current cell and move to the next one in JupyterLab?
A: Press `Shift+Enter`; `Ctrl+Enter` runs the cell but keeps focus on it, and `Alt+Enter` runs it and inserts a new cell below.

Q: What does the number in `In [5]:` next to a cell mean?
A: It is the execution count, showing the order in which cells were run during the current kernel session; a cell run later has a higher number regardless of its position in the notebook.

Q: What does `In [*]:` indicate?
A: The cell is currently running (or queued) and the kernel has not yet finished executing it.

Q: What is the "kernel" in a Jupyter notebook?
A: The separate Python process that actually executes your cell code and holds all variables and imports in memory; the notebook UI just sends code to it and displays results.

Q: What is notebook "hidden state"?
A: Variables, imports, and definitions that still live in the kernel's memory from cells you ran earlier (possibly out of order or since deleted), so the notebook's behaviour no longer matches the code you can see top-to-bottom.

Q: Why is running cells out of order dangerous?
A: Later cells may depend on state created by cells run in a different order, so the notebook works for you but is not reproducible; re-running top-to-bottom on a fresh kernel can fail or give different results.

Q: What does "Restart Kernel and Run All Cells" do and why is it important?
A: It clears all in-memory state and executes every cell from top to bottom in order; it is the gold-standard check that your notebook is reproducible and free of hidden-state surprises.

Q: You deleted a cell that defined `nav_df`, but later cells still use `nav_df` without error — why?
A: The variable is still in the kernel's memory from when the deleted cell ran; restart-and-run-all would expose the now-missing definition as a `NameError`.

Q: How do you import pandas by convention?
A: `import pandas as pd`.

Q: What is a pandas `DataFrame`?
A: A 2-D labelled table with rows (indexed) and named columns, where each column is a `Series` and can have its own dtype.

Q: What is a pandas `Series`?
A: A 1-D labelled array (one column or one row of data) with an index, holding values of a single dtype.

Q: What is the DataFrame "index"?
A: The row labels of the DataFrame; by default a `RangeIndex` (0, 1, 2, …) but it can be set to a meaningful column such as an ISIN or a date.

Q: How do you read a CSV file into a DataFrame?
A: `pd.read_csv("file.csv")`.

Q: How do you read a Parquet file into a DataFrame?
A: `pd.read_parquet("file.parquet")`.

Q: What is Parquet and why prefer it over CSV for data pipelines?
A: Parquet is a columnar, compressed binary file format that stores dtypes and is far faster and smaller to read/write than CSV, which is plain text with no type information.

Q: When you `read_csv`, why might a numeric column come in as `object` (string) dtype?
A: The CSV is text, so values like `"1,234.56"`, thousands separators, currency symbols, or stray non-numeric entries force pandas to fall back to `object`; you must clean or specify the dtype.

Q: What does the `dtype` of a column tell you?
A: The data type of its values, e.g. `int64`, `float64`, `object` (usually strings), `bool`, `datetime64[ns]`, or `category`.

Q: How do you inspect the dtype of every column at once?
A: `df.dtypes`, or `df.info()` for dtypes plus non-null counts and memory usage.

Q: Why does Parquet preserve dtypes but CSV does not?
A: Parquet stores the schema (column types) in the file metadata, whereas CSV is just text and pandas must re-infer types on every read.

Q: How do you tell `read_csv` to parse a column as dates?
A: Pass `parse_dates=["trade_date"]` (a list of column names) so those columns come in as `datetime64[ns]` instead of strings.

Q: Why parse dates at read time rather than leaving them as strings?
A: A real `datetime64` dtype enables date arithmetic, sorting, filtering by range, resampling, and `.dt` accessors; string dates sort lexicographically and cannot do any of that.

Q: An EMT file's `NAV_Date` column reads in as `object` even though it looks like dates — what is the likely cause and fix?
A: The date format was not recognised (e.g. `DD/MM/YYYY`); use `pd.to_datetime(df["NAV_Date"], format="%d/%m/%Y")` or pass an explicit `date_format`/`format` to `read_csv`.

Q: What does `df.shape` return?
A: A tuple `(n_rows, n_columns)` giving the dimensions of the DataFrame.

Q: What is the difference between `df.loc[]` and `df.iloc[]`?
A: `loc` selects by label (row/column names, inclusive of the end label), while `iloc` selects by integer position (0-based, end-exclusive like normal Python slicing).

Q: How do you select the column `nav` as a Series?
A: `df["nav"]` or `df.loc[:, "nav"]`.

Q: How do you select multiple columns as a DataFrame?
A: Pass a list: `df[["isin", "nav", "nav_date"]]`.

Q: How do you select the row labelled `LU0123456789` and the `nav` column with `loc`?
A: `df.loc["LU0123456789", "nav"]` (assuming the ISIN is the index).

Q: How do you select the first 3 rows and first 2 columns by position?
A: `df.iloc[0:3, 0:2]`.

Q: With `df.loc["2026-01-01":"2026-01-31"]`, is the end label included?
A: Yes — `loc` slicing is inclusive of both endpoints, unlike `iloc` and normal Python slicing which exclude the stop.

Q: What is a boolean mask in pandas?
A: A boolean Series (one `True`/`False` per row) used to filter rows; `df[mask]` keeps only rows where the mask is `True`.

Q: How do you filter rows where `nav` is greater than 100?
A: `df[df["nav"] > 100]`, or equivalently `df.loc[df["nav"] > 100]`.

Q: How do you combine two conditions in a boolean mask?
A: Use `&` (and), `|` (or), `~` (not) with each condition in parentheses, e.g. `df[(df["nav"] > 100) & (df["currency"] == "EUR")]`.

Q: Why must you use `&`/`|` instead of `and`/`or` in pandas masks?
A: `&`/`|` are element-wise operators on Series, while Python's `and`/`or` try to evaluate the whole Series as a single boolean and raise `ValueError: The truth value of a Series is ambiguous`.

Q: Why are parentheses required around each condition when combining masks?
A: Because `&` and `|` have higher operator precedence than comparison operators, so without parentheses pandas evaluates the bitwise op first and errors or gives wrong results.

Q: How do you select rows whose `currency` is in a set of values?
A: `df[df["currency"].isin(["EUR", "USD", "GBP"])]`.

Q: What is the "chained assignment" / `SettingWithCopyWarning` problem?
A: When you write `df[mask]["col"] = value`, pandas may assign to a temporary copy rather than the original DataFrame, so the change silently does not stick and you get a warning.

Q: What is the correct way to set values on filtered rows?
A: Use a single `loc` call: `df.loc[mask, "col"] = value`, which assigns in place on the original DataFrame.

Q: How do you set the `status` column to `"stale"` for all rows where `nav_date` is older than a cutoff?
A: `df.loc[df["nav_date"] < cutoff, "status"] = "stale"`.

Q: When you slice a DataFrame, how do you guarantee an independent copy you can safely modify?
A: Call `.copy()`, e.g. `sub = df[df["currency"] == "EUR"].copy()`, so later assignments don't trigger `SettingWithCopyWarning` and don't touch the original.

Q: What is the split-apply-combine pattern?
A: A three-step workflow: split rows into groups by some key, apply an aggregation/transformation to each group, then combine the results into a new structure — implemented by `groupby`.

Q: How do you compute the mean `nav` per `currency`?
A: `df.groupby("currency")["nav"].mean()`.

Q: How do you group by more than one column?
A: Pass a list of keys: `df.groupby(["fund_id", "currency"])`, which produces a `MultiIndex` on the result.

Q: How do you apply different aggregations to different columns in one call?
A: Use `.agg()` with a dict, e.g. `df.groupby("fund_id").agg({"nav": "mean", "shares": "sum"})`.

Q: How do you produce named aggregation columns with `groupby`?
A: Use named aggregation: `df.groupby("fund_id").agg(avg_nav=("nav", "mean"), total_shares=("shares", "sum"))`.

Q: After `df.groupby("currency")["nav"].mean()`, what is the index of the result?
A: The unique `currency` values become the index of the resulting Series.

Q: How do you turn a `groupby` result's group keys back into regular columns?
A: Call `.reset_index()`, or pass `as_index=False` to `groupby`.

Q: How do you count the number of rows in each group?
A: `df.groupby("fund_id").size()`, which counts all rows including NaNs, unlike `.count()` which counts non-null values per column.

Q: What is the difference between `size()` and `count()` on a groupby?
A: `size()` returns the total number of rows per group (including NaN), while `count()` returns the number of non-null values per column per group.

Q: How would you find, per fund, the latest NAV date?
A: `df.groupby("fund_id")["nav_date"].max()`.

Q: What does `df.merge(other, on="isin")` do?
A: Joins the two DataFrames on matching `isin` values, combining columns from both where the keys match (an inner join by default).

Q: What are the `how` options for `merge` and what do they mean?
A: `inner` (only matching keys), `left` (all left rows), `right` (all right rows), `outer` (all keys from both); the default is `inner`.

Q: What is "row multiplication" (fan-out) in a merge?
A: If a key matches multiple rows in the other table, each left row is duplicated once per match, so the result can have more rows than either input — a common cause of inflated sums.

Q: You merged transactions to a fund reference table and your total share count doubled — what likely happened?
A: The reference table had duplicate keys (e.g. two rows per ISIN), causing a many-to-many fan-out that duplicated transaction rows; deduplicate the reference or check for duplicate keys first.

Q: How can `merge` help you catch unexpected fan-out or missing matches?
A: Pass `validate="one_to_one"` / `"one_to_many"` to assert the key relationship (it raises if violated), and `indicator=True` to add a `_merge` column showing `left_only`/`right_only`/`both`.

Q: What does `indicator=True` add to a merge result?
A: A categorical `_merge` column labelling each row as `left_only`, `right_only`, or `both`, useful for auditing which keys matched.

Q: How do you merge when the key columns have different names in each table?
A: Use `left_on` and `right_on`, e.g. `df.merge(other, left_on="security_id", right_on="isin")`.

Q: What is the difference between `merge` and `concat`?
A: `merge` joins tables horizontally on key values (like a SQL join), while `pd.concat` stacks DataFrames vertically (more rows) or horizontally by index alignment without matching on a key.

Q: How do you stack two DataFrames with the same columns into one?
A: `pd.concat([df1, df2], ignore_index=True)` to append rows and renumber the index.

Q: What does `df["nav_date"].dt.year` give you?
A: A Series of the year extracted from each datetime value, via the `.dt` accessor (similarly `.dt.month`, `.dt.day`, `.dt.dayofweek`).

Q: What must a column's dtype be before you can use the `.dt` accessor?
A: It must be a datetime type (`datetime64[ns]`); on an `object`/string column `.dt` raises an `AttributeError`.

Q: What does `resample` do?
A: Groups time-series rows into regular time buckets (e.g. daily, monthly) and aggregates each bucket — like a `groupby` over time, requiring a datetime index or `on=`.

Q: How do you compute month-end average NAV from a daily series?
A: Set the date as the index (or use `on="nav_date"`) and call `df.resample("ME")["nav"].mean()` (`"ME"` = month-end frequency).

Q: What does the resample rule `"D"` vs `"W"` vs `"ME"` mean?
A: `"D"` = daily buckets, `"W"` = weekly, `"ME"` = month-end; the rule string sets the bucket frequency.

Q: What is required for `resample` to work directly on a DataFrame?
A: A `DatetimeIndex` (set via `set_index` or `parse_dates`), or pass the datetime column with `on="nav_date"`.

Q: How do you set a column as the index?
A: `df = df.set_index("nav_date")`.

Q: What is `NaN` in pandas?
A: "Not a Number" — the standard missing-value marker for floats; pandas also uses `NaT` for missing datetimes and `None`/`pd.NA` for missing objects.

Q: How do you count missing values per column?
A: `df.isna().sum()`, which sums the boolean `True`s (missing) for each column.

Q: What does `df.dropna()` do by default?
A: Drops every row that contains at least one `NaN` in any column; use `subset=` to limit which columns count, or `how="all"` to drop only fully-empty rows.

Q: What does `df.fillna(0)` do?
A: Replaces all `NaN` values with `0` (or another specified value), keeping the rows in place.

Q: When should you choose `fillna` over `dropna`?
A: Use `fillna` when missing values have a meaningful default and you must keep the rows; use `dropna` when rows with missing data are invalid or unusable for the analysis.

Q: What is forward-fill and how do you do it?
A: `df.ffill()` propagates the last valid value forward to fill gaps — useful for carrying a NAV over non-trading days; `bfill()` does the reverse.

Q: Why can summing a column with `NaN`s still work but a row-count be misleading?
A: pandas aggregations like `sum`/`mean` skip `NaN` by default, so they return a value, but the count of contributing rows is smaller than the total row count — check `count()` to see how many values were actually used.

Q: What is the difference between `==` and `.isna()` when checking for missing values?
A: `NaN == NaN` is `False`, so `df["col"] == np.nan` never matches; you must use `df["col"].isna()` (or `.isnull()`) to detect missing values.

Q: How do you replace specific values (not just NaN), e.g. `"N/A"` strings with missing?
A: `df["col"].replace("N/A", pd.NA)`, or pass `na_values=["N/A"]` to `read_csv` so they import as missing.

Q: How do you make a quick line chart of NAV over time from a DataFrame?
A: With a datetime index, `df["nav"].plot()` (uses matplotlib) draws a line chart; in JupyterLab it renders inline automatically.

Q: What does `df.plot(kind="bar")` produce versus `kind="line"`?
A: `kind="bar"` draws a bar chart (good for categorical comparisons like NAV per fund), while `kind="line"` draws a line chart (good for trends over time).

Q: Why might your chart not appear inline in a notebook, and what fixes it?
A: It usually appears automatically in JupyterLab; if not, ensure matplotlib is installed and import it (`import matplotlib.pyplot as plt`), and call `plt.show()` if needed.

Q: How do you write a DataFrame to a Parquet file?
A: `df.to_parquet("output.parquet")` (requires `pyarrow` or `fastparquet`).

Q: How do you write a DataFrame to a database table?
A: `df.to_sql("table_name", con=engine, if_exists="append", index=False)`, where `engine` is a SQLAlchemy connection.

Q: What does the `if_exists` argument of `to_sql` control?
A: What to do if the target table already exists: `"fail"` (default, error), `"replace"` (drop and recreate), or `"append"` (add rows).

Q: Why pass `index=False` to `to_sql` (and often `to_csv`)?
A: To avoid writing the DataFrame's index as an extra unnamed column in the table/file when the index is just a default `RangeIndex`.

Q: Why prefer `to_parquet` over `to_csv` for intermediate pipeline outputs?
A: Parquet keeps dtypes, is compressed and columnar (smaller, faster), and round-trips numbers and dates exactly, whereas CSV loses types and bloats on disk.

Q: What is a `category` dtype and when is it useful?
A: A memory-efficient dtype for columns with few distinct repeated values (e.g. `currency`, `fund_id`); it stores integer codes plus a small lookup, saving memory and speeding up groupby.

Q: How do you convert a column to `category` dtype?
A: `df["currency"] = df["currency"].astype("category")`.

Q: How do you change a column's dtype, e.g. string to float?
A: `df["nav"] = df["nav"].astype("float64")`, after cleaning any non-numeric characters; or use `pd.to_numeric(df["nav"], errors="coerce")` to turn bad values into `NaN`.

Q: What does `pd.to_numeric(s, errors="coerce")` do?
A: Converts a Series to numeric, turning any unparseable values into `NaN` instead of raising — handy for messy text columns like prices with stray symbols.

Q: How do you remove duplicate rows?
A: `df.drop_duplicates()`, optionally with `subset=["isin", "nav_date"]` to define duplicates by specific columns and `keep="first"/"last"`.

Q: How do you check for duplicate keys before a merge?
A: `df["isin"].duplicated().any()` returns `True` if any ISIN repeats; `df["isin"].value_counts()` shows which keys repeat and how often.

Q: How do you rename columns?
A: `df.rename(columns={"NAV_Date": "nav_date", "NAV": "nav"})`, which returns a new DataFrame with the renamed columns.

Q: How do you sort a DataFrame by a column?
A: `df.sort_values("nav_date")`, with `ascending=False` for descending and a list of columns for multi-key sorting.

Q: How do you create a new column from a calculation on existing columns?
A: Assign it directly, e.g. `df["market_value"] = df["nav"] * df["shares"]`, which is vectorised across all rows.

Q: Why prefer vectorised column operations over looping row by row?
A: Vectorised operations run in fast compiled C code over the whole array, while Python row loops (or `.iterrows()`) are far slower and harder to read.

Q: How do you get the unique values and their frequencies in a column?
A: `df["currency"].unique()` lists distinct values; `df["currency"].value_counts()` gives each value with its count, sorted descending.

Q: How do you pivot long data to wide, e.g. one column per currency?
A: `df.pivot_table(index="nav_date", columns="currency", values="nav", aggfunc="mean")`.

Q: Your `read_csv` fails with a parsing/tokenizing error about the wrong number of fields — what do you check first?
A: The delimiter (use `sep=";"` for European CSVs), quoting, or embedded delimiters in fields; also check for a ragged header or extra rows needing `skiprows`.

Q: A European EMT CSV uses `;` as separator and `,` as the decimal mark — how do you read it correctly?
A: `pd.read_csv("emt.csv", sep=";", decimal=",")` so fields split on `;` and numbers parse with a comma decimal.

Q: Your notebook gives different results than a colleague's despite identical code — what is the most likely cause?
A: Hidden state from running cells out of order or stale variables; both should restart the kernel and run all cells to compare on a clean, reproducible run.

Q: A column of ISINs lost its leading zeros after `read_csv` — why, and how do you prevent it?
A: pandas inferred it as a numeric dtype and dropped leading zeros; force it to text with `dtype={"isin": "string"}` (or `dtype=str`) at read time.
