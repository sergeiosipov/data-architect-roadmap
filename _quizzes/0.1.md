# Phase 0 · 0.1 Linux, the command line & shell — 100 self-test questions

Q: What is WSL2 and why would a Windows-based data engineer use it?
A: WSL2 (Windows Subsystem for Linux 2) runs a real Linux kernel in a lightweight VM on Windows, letting you run Ubuntu and Linux tooling natively; it lets you develop in the same Linux environment your production fund-data pipelines will run on.

Q: What command installs Ubuntu under WSL2 from a fresh Windows machine?
A: `wsl --install` (optionally `wsl --install -d Ubuntu`) installs WSL2 with Ubuntu as the default distribution.

Q: How do you check which WSL version a distribution is using?
A: Run `wsl -l -v` (list verbose) in PowerShell; the `VERSION` column shows `1` or `2` for each installed distribution.

Q: Inside WSL2, where do you find your Windows `C:` drive?
A: It is mounted under `/mnt/c`, so `C:\Users\senio` is `/mnt/c/Users/senio` from the Linux side.

Q: Why is it faster to keep project files in the Linux filesystem (`/home`) rather than `/mnt/c` under WSL2?
A: Cross-filesystem access between the Linux VM and Windows (`/mnt/c`) has high I/O overhead, so reading/writing many files (e.g. a folder of EMT files) is much slower than working under the native Linux ext4 filesystem.

Q: What does the Linux filesystem hierarchy directory `/etc` hold?
A: System-wide configuration files (e.g. `/etc/passwd`, `/etc/crontab`, `/etc/hosts`); it is for config, not user data or binaries.

Q: What is stored under `/home`?
A: Per-user home directories, e.g. `/home/senio`, holding that user's personal files and dotfile configuration; `~` is shorthand for the current user's home.

Q: What kind of data lives under `/var`?
A: Variable data that changes at runtime — logs (`/var/log`), spool/queue files, caches, and mail; you would check `/var/log` to troubleshoot a failed nightly NAV job.

Q: What is `/tmp` used for and what is its key caveat?
A: `/tmp` is for temporary files; it is world-writable and is typically cleared on reboot, so never store anything you need to keep there.

Q: In `ls -l` output, what do the three triplets of `rwx` represent?
A: The permission bits for owner, group, and others respectively, each showing read (`r`), write (`w`), and execute (`x`) permissions.

Q: In `-rw-r--r--`, what is the leading `-` and what are the permissions?
A: The leading `-` means it is a regular file (a `d` would mean directory); owner has read+write, group has read, others have read.

Q: What does the execute (`x`) bit mean on a directory versus a file?
A: On a file `x` means it can be run as a program; on a directory `x` means you can enter it (`cd` into it) and access files within by name.

Q: What is the numeric (octal) value of permissions `rwxr-xr--`?
A: `754` — owner `rwx`=7, group `r-x`=5, others `r--`=4.

Q: How would you make a script `load_nav.sh` executable for everyone?
A: `chmod +x load_nav.sh`, or explicitly `chmod 755 load_nav.sh`.

Q: What does `chmod 600 id_rsa` accomplish and why does it matter for SSH?
A: It gives the owner read+write and no one else any access; SSH refuses to use a private key file that is readable by group or others.

Q: What does `chown` do and what is the syntax to change both owner and group?
A: `chown` changes file ownership; `chown user:group file` sets owner and group, e.g. `chown senio:dataeng nav.csv`.

Q: When do you typically need `sudo`?
A: When performing actions requiring root/administrator privileges, such as installing packages, editing files under `/etc`, or writing outside your home directory.

Q: Why is running everything as root (or blanket `sudo`) discouraged?
A: It removes the safety net of permission checks, so a typo or malicious script can damage the whole system; least-privilege limits blast radius.

Q: What does `chmod -R 755 reports/` do?
A: Recursively sets `755` permissions on `reports/` and everything inside it; `-R` means recurse into subdirectories.

Q: What does `pwd` print?
A: The present working directory — the absolute path of the directory you are currently in.

Q: What is the difference between an absolute path and a relative path?
A: An absolute path starts from root `/` (e.g. `/home/senio/data`); a relative path is interpreted from the current directory (e.g. `data/nav.csv`).

Q: What do `.` and `..` refer to?
A: `.` is the current directory and `..` is the parent directory; `cd ..` moves up one level.

Q: What does `cd` with no arguments do?
A: It returns you to your home directory (`$HOME`), equivalent to `cd ~`.

Q: What does `cd -` do?
A: It switches to the previous working directory you were in (the value of `$OLDPWD`).

Q: What does `ls -la` show that plain `ls` does not?
A: `-l` gives the long format (permissions, owner, size, date); `-a` includes hidden dotfiles like `.bashrc`.

Q: How do you list files sorted by modification time, newest first?
A: `ls -lt` (add `-r` to reverse to oldest first); useful for finding the latest delivered EMT file in a drop folder.

Q: What does the `|` (pipe) operator do?
A: It connects the standard output of one command to the standard input of the next, e.g. `cat nav.csv | grep LU` filters lines containing `LU`.

Q: What is the difference between `>` and `>>`?
A: `>` redirects stdout to a file, overwriting it; `>>` appends to the file, preserving existing contents.

Q: What does `2>` redirect?
A: File descriptor 2, standard error (stderr); e.g. `./job.sh 2> errors.log` sends only error output to the file.

Q: How do you redirect both stdout and stderr to the same file?
A: `command > out.log 2>&1` (or in bash `command &> out.log`); the `2>&1` sends stderr to wherever stdout currently points.

Q: What does the order matter in `command 2>&1 > file` versus `command > file 2>&1`?
A: `> file 2>&1` sends both to the file; `2>&1 > file` sends stderr to the terminal (stdout's original target) and only stdout to the file, because redirections are applied left to right.

Q: What is `/dev/null` and a common use for it?
A: A special device that discards everything written to it; `command 2>/dev/null` suppresses error messages, and `> /dev/null 2>&1` discards all output.

Q: What are the three standard file descriptors and their numbers?
A: stdin (0), stdout (1), and stderr (2).

Q: What does `tee` do in a pipeline?
A: It reads stdin and writes to both stdout and one or more files, e.g. `./job.sh | tee run.log` shows output on screen and saves it.

Q: What does `grep "LU" isin_list.txt` do?
A: It prints every line of `isin_list.txt` containing the substring `LU` (e.g. Luxembourg-domiciled ISINs).

Q: What does `grep -i` do?
A: Case-insensitive matching, so `grep -i nav` matches `NAV`, `nav`, and `Nav`.

Q: How do you grep recursively through a directory of files?
A: `grep -r "ISIN" data/` searches all files under `data/`; add `-l` to print only the names of matching files.

Q: How would you count how many lines in `holdings.csv` contain a Luxembourg ISIN (prefix `LU`)?
A: `grep -c "^LU" holdings.csv` if the ISIN starts the line, or `grep -c "LU" holdings.csv` to count any line containing `LU`.

Q: What does the `-E` flag give `grep`?
A: Extended regular expressions, allowing `+`, `?`, `|`, and `()` without backslash escaping, e.g. `grep -E "NAV|TNA"`.

Q: What does `sed 's/old/new/'` do?
A: It performs a substitution, replacing the first occurrence of `old` with `new` on each line; output goes to stdout by default.

Q: How do you make `sed` replace all occurrences on each line, not just the first?
A: Add the global flag: `sed 's/old/new/g'`.

Q: How do you edit a file in place with `sed`?
A: `sed -i 's/;/,/g' emt.csv` modifies the file directly; on GNU sed `-i` works, and a backup can be kept with `-i.bak`.

Q: How would you convert a semicolon-delimited EMT file to comma-delimited with `sed`?
A: `sed 's/;/,/g' emt.csv > emt_comma.csv` (assuming no semicolons appear inside quoted fields).

Q: What does `awk '{print $2}'` do?
A: It prints the second whitespace-separated field of each line; `$0` is the whole line, `$1` the first field, etc.

Q: How do you tell `awk` the fields are separated by commas?
A: Use `-F,` (or `-F','`), e.g. `awk -F, '{print $1}' nav.csv` prints the first comma-separated column.

Q: How would you use `awk` to print the ISIN and NAV columns (say fields 1 and 4) of a CSV?
A: `awk -F, '{print $1, $4}' nav.csv`; the comma in `print` inserts the output field separator (a space by default).

Q: How do you filter rows in `awk` where the NAV (field 4) exceeds 100?
A: `awk -F, '$4 > 100' nav.csv` prints rows whose fourth field is numerically greater than 100.

Q: How would you sum a numeric column with `awk`?
A: `awk -F, '{sum += $4} END {print sum}' nav.csv` accumulates field 4 and prints the total after the last line.

Q: What is the difference between `grep`, `sed`, and `awk` at a high level?
A: `grep` finds/filters lines by pattern, `sed` does stream edits like search-and-replace, and `awk` is a small language for field-based extraction and computation.

Q: What does the `$PATH` environment variable control?
A: The ordered list of directories the shell searches for an executable when you type a command; "command not found" means the binary is in none of those directories.

Q: How do you print the value of an environment variable?
A: `echo $VAR` (e.g. `echo $PATH`) or `printenv VAR`.

Q: How do you set an environment variable for the current shell session only?
A: `export VAR=value`, e.g. `export DB_HOST=localhost`; it persists for the session and child processes but not after closing the shell.

Q: What is the difference between `VAR=value` and `export VAR=value`?
A: Plain `VAR=value` sets a shell variable visible only to the current shell; `export` marks it for inheritance by child processes (it becomes an environment variable).

Q: How do you append a directory to `$PATH` without losing the existing entries?
A: `export PATH="$PATH:/new/dir"` (or prepend with `export PATH="/new/dir:$PATH"` to give it priority).

Q: Where would you put `export` lines so they apply to every new interactive shell?
A: In your shell startup file, typically `~/.bashrc` (or `~/.profile`/`~/.bash_profile` for login shells).

Q: After editing `~/.bashrc`, how do you apply the changes without logging out?
A: Run `source ~/.bashrc` (or `. ~/.bashrc`) to re-read the file into the current shell.

Q: How do you find which executable will run for a given command name?
A: `which command` (or `type command`) shows the full path of the binary that would be executed based on `$PATH`.

Q: How do you unset an environment variable?
A: `unset VAR` removes it from the current shell environment.

Q: What is SSH and what is it used for?
A: Secure Shell — an encrypted protocol for logging into and running commands on remote machines, and for secure file transfer (scp/sftp) and tunneling.

Q: What is the difference between an SSH public key and private key?
A: The private key (e.g. `~/.ssh/id_ed25519`) stays secret on your machine; the public key (`.pub`) is copied to servers, which use it to verify you hold the matching private key.

Q: What command generates a modern SSH key pair?
A: `ssh-keygen -t ed25519 -C "you@example.com"` creates an Ed25519 key pair, recommended over older RSA.

Q: How do you copy your public key to a remote server's authorized keys?
A: `ssh-copy-id user@host` appends your public key to the server's `~/.ssh/authorized_keys`; otherwise paste it there manually.

Q: What is the SSH agent and why use it?
A: `ssh-agent` is a background process that holds your decrypted private keys in memory so you enter your passphrase once per session instead of on every connection.

Q: How do you add a key to the running SSH agent?
A: `ssh-add ~/.ssh/id_ed25519` (start the agent first with `eval "$(ssh-agent -s)"` if it is not running).

Q: What is the purpose of the `~/.ssh/config` file?
A: It stores per-host connection settings (hostname, user, port, identity file) so you can connect with a short alias, e.g. `ssh prod-sftp`.

Q: What does a "Permissions 0644 for 'id_ed25519' are too open" SSH error mean and how do you fix it?
A: SSH refuses to use a private key that others can read; fix with `chmod 600 ~/.ssh/id_ed25519`.

Q: What is `known_hosts` and why does SSH warn about a changed host key?
A: `~/.ssh/known_hosts` records server fingerprints; a changed key triggers a warning because it could indicate a man-in-the-middle attack (or a legitimately rebuilt server).

Q: What does a shebang line like `#!/usr/bin/env bash` do?
A: It tells the OS which interpreter to run the script with; using `env bash` finds bash via `$PATH` rather than hardcoding `/bin/bash`.

Q: Why must the shebang be the very first line of a script?
A: The kernel only checks the first two bytes (`#!`) of the file to choose the interpreter; if anything precedes it, the shebang is ignored.

Q: In a bash script, what do `$1`, `$2`, etc. mean?
A: They are positional parameters — the first, second, etc. command-line arguments passed to the script.

Q: What is the difference between `$@` and `$*`?
A: Both expand to all positional arguments, but `"$@"` expands each argument as a separate quoted word (correct for preserving spaces), while `"$*"` joins them into a single word.

Q: What does `$#` give you in a script?
A: The number of positional arguments passed to the script.

Q: What is an exit code and what does `0` conventionally mean?
A: An exit code is the integer a command returns on completion; `0` means success and any non-zero value (1–255) signals an error.

Q: How do you check the exit code of the command you just ran?
A: Inspect `$?`, e.g. `echo $?` prints the exit status of the last command.

Q: How do you make a script exit with a specific failure code?
A: Use `exit N`, e.g. `exit 1` to signal a generic error to the caller (such as a cron job or orchestrator).

Q: What does `set -e` do in a bash script?
A: It makes the script exit immediately if any command returns a non-zero status, so failures do not silently continue.

Q: What does `set -u` do?
A: It treats the use of any unset variable as an error and exits, catching typos like `$INPTU_FILE` instead of `$INPUT_FILE`.

Q: What does `set -o pipefail` do?
A: It makes a pipeline return the exit status of the first failing command rather than only the last command's status, so `grep x file | sort` fails if `grep` fails.

Q: Why is `set -euo pipefail` a recommended header for production bash scripts?
A: Together they make the script fail fast on errors (`-e`), unset variables (`-u`), and broken pipelines (`pipefail`), which prevents a partially-failed NAV load from looking successful.

Q: What is the basic syntax of an `if` statement in bash?
A: `if condition; then ...; fi`, e.g. `if [ -f nav.csv ]; then echo found; fi`; optional `elif` and `else` branches are allowed.

Q: What is the difference between `[` and `[[` in bash conditionals?
A: `[` is the older POSIX `test` command; `[[ ]]` is a bash keyword with safer behavior — it supports `&&`, `||`, pattern matching, and avoids word-splitting pitfalls.

Q: How do you write a `for` loop over files in bash?
A: `for f in *.csv; do echo "$f"; done` iterates over each matching filename; always quote `"$f"` to handle spaces.

Q: How would you loop over each EMT file in a folder and run a load script on it?
A: `for f in emt/*.csv; do ./load_emt.sh "$f"; done` runs `load_emt.sh` once per file.

Q: Why should you quote variables like `"$f"` in scripts?
A: Unquoted variables undergo word-splitting and glob expansion, so a filename with spaces or special characters would break into multiple arguments; quoting preserves it as one value.

Q: What is command substitution and how do you write it?
A: It captures a command's output into a value using `$(command)` (preferred) or backticks, e.g. `today=$(date +%F)`.

Q: How do you provide a default value for a possibly-unset variable?
A: Parameter expansion `${VAR:-default}` yields `default` if `VAR` is unset or empty, e.g. `dir=${DATA_DIR:-/data}`.

Q: What does `apt update` do versus `apt upgrade`?
A: `apt update` refreshes the local package index from the repositories; `apt upgrade` actually installs newer versions of already-installed packages.

Q: Why must you usually prefix `apt` commands with `sudo`?
A: Installing, upgrading, or removing system packages modifies system directories and the package database, which requires root privileges.

Q: How do you install a package with `apt`?
A: `sudo apt install <package>`, e.g. `sudo apt install jq` to install the JSON-processing tool.

Q: How do you search for an available package and inspect one?
A: `apt search <term>` lists matching packages and `apt show <package>` displays details like version and dependencies.

Q: How do you remove a package, and what is the difference between `remove` and `purge`?
A: `sudo apt remove <package>` deletes the package but keeps its config files; `sudo apt purge <package>` removes the package and its system config files too.

Q: What is the difference between `apt` and `dpkg`?
A: `dpkg` is the low-level tool that installs individual `.deb` files without resolving dependencies; `apt` is the high-level front end that fetches packages and resolves dependencies from repositories.

Q: What is cron used for?
A: Scheduling commands or scripts to run automatically at fixed times or intervals, e.g. a nightly job that ingests NAV files at 02:00.

Q: How do you edit your personal crontab?
A: `crontab -e` opens your user crontab in an editor; `crontab -l` lists current entries and `crontab -r` removes them all.

Q: What do the five time fields of a crontab entry represent, in order?
A: Minute (0–59), hour (0–23), day of month (1–31), month (1–12), and day of week (0–7, where both 0 and 7 are Sunday).

Q: What schedule does the cron line `0 2 * * 1-5` represent?
A: 02:00 every Monday through Friday — useful for a NAV load that only runs on EU business days.

Q: What does `*/15 * * * *` mean in cron?
A: Run every 15 minutes (at minutes 0, 15, 30, 45 of every hour).

Q: Why do cron jobs often fail with "command not found" even when the script works in your terminal?
A: Cron runs with a minimal environment and a very limited `$PATH`, so you must use absolute paths to binaries or set `PATH` explicitly inside the crontab or script.

Q: How do you capture the output and errors of a cron job for troubleshooting?
A: Redirect in the crontab line, e.g. `0 2 * * * /opt/jobs/load_nav.sh >> /var/log/nav.log 2>&1`, so both stdout and stderr are appended to a log.

Q: A nightly cron job that loads transfer-agency files silently produces no data — what should you check first?
A: Check the job's log/output and exit code, then verify cron's environment: confirm absolute paths, the `$PATH`, file permissions on the input directory, and that the script is executable.
