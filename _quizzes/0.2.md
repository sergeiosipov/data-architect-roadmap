# Phase 0 · 0.2 Programming from zero: Python — 100 self-test questions

Q: What is a Python variable, and how do you create one?
A: A variable is a name bound to a value; you create it by assignment, e.g. `nav = 12.34`, with no separate declaration or type annotation required.

Q: Name Python's four most common built-in scalar types and a value of each.
A: `int` (e.g. `42`), `float` (e.g. `12.34`), `str` (e.g. `"ISIN"`), and `bool` (`True`/`False`).

Q: What does the `type()` function return, and why is it useful when debugging?
A: It returns the class/type of an object, e.g. `type(nav)` -> `<class 'float'>`; it's useful to confirm a value isn't a string when arithmetic misbehaves.

Q: What is the difference between `==` and `is` in Python?
A: `==` compares values for equality, while `is` checks whether two names refer to the exact same object in memory; use `==` for value comparisons and `is` mainly for `is None`.

Q: Why should you write `if x is None:` rather than `if x == None:`?
A: `None` is a singleton, so identity (`is`) is the correct and faster check, and it avoids surprises from objects that override `__eq__`.

Q: What is the difference between `5 / 2` and `5 // 2` in Python 3?
A: `5 / 2` is true division giving `2.5` (a `float`), while `5 // 2` is floor division giving `2` (an `int`).

Q: What does `int("0012")` return, and what does `int("12.0")` do?
A: `int("0012")` returns `12`, but `int("12.0")` raises `ValueError` because `int()` won't parse a decimal-point string directly.

Q: Why can converting a NAV string like `"1,234.56"` to a number fail, and how do you handle it?
A: `float("1,234.56")` raises `ValueError` because of the thousands separator; strip or replace separators first (e.g. `value.replace(",", "")`) before converting.

Q: What is an f-string and why is it preferred for building messages?
A: An f-string is a string literal prefixed with `f` that interpolates expressions in braces, e.g. `f"Fund {isin} NAV {nav:.4f}"`; it's concise, readable, and supports format specs.

Q: What does the format spec `:.4f` do inside an f-string?
A: It formats a number as a fixed-point decimal with exactly 4 digits after the point, which suits NAV values commonly quoted to 4 decimals.

Q: What is the difference between mutable and immutable types, with one example each?
A: Mutable objects can be changed in place (e.g. `list`, `dict`, `set`); immutable objects cannot (e.g. `int`, `str`, `tuple`), so "changing" them creates a new object.

Q: What value does an empty string, empty list, `0`, and `None` have in a boolean context?
A: They are all falsy, so `if not value:` is true for each; use this carefully because `0` and `""` are valid data that may not mean "missing".

Q: Write an `if/elif/else` that classifies a currency as `"EUR"`, `"USD"`, or `"other"`.
A: `if ccy == "EUR": ... elif ccy == "USD": ... else: ...`, where each branch handles one case and `else` catches the rest.

Q: How does Python delimit a code block instead of using braces?
A: By indentation; a consistent indent (conventionally 4 spaces) defines the block, and mixing tabs and spaces raises an error.

Q: What does a `for` loop iterate over, and how do you loop over a list of ISINs?
A: A `for` loop iterates over any iterable; `for isin in isins:` binds each element of the list to `isin` in turn.

Q: How do you get both the index and the value while looping over a list?
A: Use `enumerate`, e.g. `for i, row in enumerate(rows):`, optionally `enumerate(rows, start=1)` to count from 1 (handy for human-readable row numbers).

Q: What is the difference between `break` and `continue`?
A: `break` exits the enclosing loop entirely, while `continue` skips to the next iteration without running the rest of the current loop body.

Q: When would you use a `while` loop instead of a `for` loop?
A: When you don't know the iteration count in advance and loop until a condition changes, e.g. paging through an API until no more results are returned.

Q: What is a list comprehension, and rewrite `[x for x in navs if x > 0]` in words?
A: A concise expression that builds a list; this one collects each `x` from `navs` keeping only the positive values.

Q: How do you define a function that takes a row and returns a parsed fund record?
A: Use `def parse_row(row):` followed by an indented body that ends in `return record`; the function name and parameters describe its single job.

Q: What is the difference between a positional argument and a keyword argument?
A: Positional arguments are matched by order, keyword arguments by name (e.g. `f(2, ccy="EUR")`); keywords improve readability and let you skip optional parameters.

Q: What is a default parameter value, and what is the classic mutable-default pitfall?
A: A default is used when the caller omits the argument; using a mutable default like `def f(items=[])` is a trap because the same list persists across calls — use `None` and create inside instead.

Q: What does a function return if it has no `return` statement?
A: It returns `None` implicitly, so calling it for a value gives `None` rather than an error.

Q: What is the difference between `print()` and `return` in a function?
A: `print()` writes to stdout for humans, while `return` hands a value back to the caller for further use; a function meant to be composed should return, not print.

Q: Why does the plan favour small composable functions (parse, filter, summarize, write) over one big script blob?
A: Small single-purpose functions are independently readable, reusable, and unit-testable, and they make failures easier to localize than a monolithic script.

Q: What is variable scope, and can a function see variables defined outside it?
A: Scope is the region where a name is visible; a function can read enclosing/global names but assigning to a name inside makes it local unless declared `global` or `nonlocal`.

Q: What is a module in Python?
A: A module is a `.py` file whose top-level names (functions, classes, variables) can be imported and reused by other files.

Q: What is the difference between `import json` and `from json import dumps`?
A: `import json` binds the module so you call `json.dumps(...)`, while `from json import dumps` binds the name directly so you call `dumps(...)`; the first keeps the namespace explicit.

Q: What does `if __name__ == "__main__":` do?
A: It guards code so it runs only when the file is executed directly, not when it's imported as a module, keeping importable functions free of side effects.

Q: What is the standard library, and name two modules you'd use building a CSV loader.
A: It's the batteries-included set of modules shipped with Python; `csv` for parsing rows and `json` for writing the summary are typical choices.

Q: What does the `csv` module's `csv.DictReader` give you per row?
A: A dict keyed by the header column names, so you access fields like `row["ISIN"]` instead of by numeric index, which is more robust to column reordering.

Q: What is the difference between a list and a tuple?
A: Both are ordered sequences, but lists are mutable (`[]`) and tuples are immutable (`()`); use a tuple for fixed records and as dict keys.

Q: What does list slicing `rows[1:]` return for a CSV that includes a header row?
A: All rows except the first, a common idiom for skipping a header when you've read the file into a list of lines.

Q: What is a dict, and why is it the natural fit for a funds-by-ISIN lookup?
A: A dict maps keys to values with average O(1) lookup; keying funds by their unique ISIN gives instant retrieval of a fund record by its identifier.

Q: What's the difference between `d["ISIN"]` and `d.get("ISIN")` when the key is missing?
A: `d["ISIN"]` raises `KeyError`, while `d.get("ISIN")` returns `None` (or a supplied default), which is safer when a field may be absent.

Q: What does `dict.setdefault(key, [])` do, and how does it help build a counts structure?
A: It returns the existing value or inserts and returns the default if the key is missing, letting you append to per-key lists without a separate existence check.

Q: What is `collections.Counter` good for in this loader?
A: It counts hashable items efficiently, so `Counter(currencies)` directly yields counts per currency for the summary JSON.

Q: What is a set, and what two properties make it distinct from a list?
A: A set is an unordered collection of unique, hashable elements; it removes duplicates automatically and offers fast membership tests.

Q: How would you find ISINs present in file A but not in file B using sets?
A: Convert both to sets and subtract: `set(a) - set(b)` yields the ISINs only in A.

Q: Why is checking `isin in my_set` faster than `isin in my_list` for large data?
A: Set membership is average O(1) via hashing, whereas list membership is O(n) because it scans element by element.

Q: Can a list be a dict key or a set element? Why or why not?
A: No — lists are unhashable because they're mutable; use a tuple instead, which is hashable and can serve as a key or set element.

Q: What does the `with open(path) as f:` pattern guarantee?
A: It opens the file in a context manager that automatically closes it when the block exits, even if an exception is raised.

Q: Why prefer `with open(...)` over a bare `open()` plus manual `close()`?
A: The context manager closes the file deterministically, preventing leaked file handles and partially written files when errors occur.

Q: Why should you pass `encoding="utf-8"` when opening text files?
A: It makes decoding explicit and portable; relying on the platform default can corrupt non-ASCII fund names or fail differently on Windows vs Linux.

Q: What does iterating directly over a file object yield?
A: One line at a time (including the trailing newline), which lets you process a large CSV line by line without loading it all into memory.

Q: What is an exception, and name two common ones a CSV loader might hit.
A: An exception is a runtime error object that interrupts normal flow; `FileNotFoundError` (missing input) and `ValueError` (bad number conversion) are typical.

Q: Write the shape of a `try/except` that catches a bad number and reports the row.
A: `try: nav = float(row["nav"]) except ValueError: raise ValueError(f"bad NAV on row {n}: {row['nav']!r}")`, turning a vague error into a located one.

Q: Why is `except Exception:` (or bare `except:`) usually a bad idea?
A: It swallows every error including bugs and `KeyboardInterrupt`-adjacent issues, hiding the real problem; catch the specific exceptions you can actually handle.

Q: What does the `finally` clause guarantee?
A: It runs whether or not an exception occurred, used for cleanup; with `with`, much of this need disappears.

Q: How do you re-raise the current exception after logging it?
A: Use a bare `raise` inside the `except` block, which preserves the original traceback rather than masking it.

Q: What does `raise ValueError("malformed row 42")` accomplish in the loader spec?
A: It fails loudly with a precise message including the row number, so a malformed transfer-agency row is diagnosable instead of silently skipped.

Q: How do you make a CLI exit non-zero with a clean one-line message instead of a raw traceback?
A: Catch the expected error at the top level, print a short message to stderr, and call `sys.exit(1)` (or `raise SystemExit("message")`).

Q: What does `raise ... from err` do?
A: It chains exceptions, showing the original cause in the traceback ("The above exception was the direct cause..."), preserving the debugging trail.

Q: What is `uv` and what role does it play in this plan?
A: `uv` is a fast Python package and project manager (from Astral) that handles environments, dependencies, and running code; it's the only sanctioned way Python runs here.

Q: What does `uv init fundcli` create?
A: A new project directory scaffold with a `pyproject.toml`, a starter source file, and version-control metadata, ready for dependencies and `uv run`.

Q: What does `uv add httpx` do?
A: It adds `httpx` as a project dependency, resolves and installs it into the project environment, and records it in `pyproject.toml` and the lockfile.

Q: What does `uv run fundcli.py --help` do that bare `python fundcli.py` does not?
A: It ensures the project's locked environment exists and is in sync first, then runs the script inside it — no manual venv activation needed.

Q: What is the `uv.lock` file and why does it matter?
A: It's the lockfile pinning exact resolved versions (and hashes) of all dependencies, so every machine reproduces the identical environment.

Q: Why is bare `pip install` banned in this plan?
A: Bare `pip` mutates whatever environment is active, isn't reproducible or lock-tracked, and easily corrupts the system Python; `uv` gives reproducible, isolated, fast installs instead.

Q: Should you commit `uv.lock` to git? Why?
A: Yes — committing the lockfile is how teammates and CI get byte-for-byte reproducible environments; it's the source of truth for versions.

Q: What is the difference between `pyproject.toml` and `uv.lock`?
A: `pyproject.toml` declares your intended dependencies and constraints, while `uv.lock` records the exact resolved versions that satisfy them.

Q: What does `uv sync` do?
A: It makes the project environment match the lockfile exactly, installing or removing packages as needed; `uv run` does this implicitly before running.

Q: How do you add a tool you only need for development, like ruff, without shipping it as a runtime dependency?
A: Add it to a dev dependency group, e.g. `uv add --dev ruff`, so it's available locally and in CI but not required by consumers.

Q: A traceback is many lines long — where is the actual error, and which line names the cause?
A: Read it bottom-up: the last line names the exception type and message, and the frames above show the call path that led there.

Q: In a traceback, what do "File ..., line N, in func" entries represent?
A: Each is a stack frame — a function call in progress — listed from outermost call at the top down to the line that actually raised at the bottom.

Q: What does `TypeError: unsupported operand type(s) for +: 'int' and 'str'` tell you?
A: You tried to add a number and a string; likely a value read from CSV is still text and needs `int()`/`float()` before arithmetic.

Q: What does `KeyError: 'ISIN'` during CSV processing usually indicate?
A: The dict (often a CSV row) has no `'ISIN'` key — commonly a header typo, wrong delimiter, or a column-name mismatch like `isin` vs `ISIN`.

Q: What does `IndentationError` mean and what's the usual cause?
A: The code's indentation is inconsistent or unexpected; common causes are mixing tabs and spaces or misaligning a block.

Q: What is the recommended first step when debugging a failure you don't understand?
A: Reproduce it minimally — strip the input and code down to the smallest example that still fails — then read the traceback bottom-up before changing anything.

Q: How can `print()` or the `breakpoint()` builtin help during debugging?
A: `print()` reveals intermediate values quickly, while `breakpoint()` drops you into an interactive debugger (`pdb`) at that line to inspect state and step through.

Q: What is the difference between a syntax error and a runtime exception?
A: A syntax error is caught before any code runs (the file won't parse), while a runtime exception occurs during execution on a particular input.

Q: What does `NameError: name 'navs' is not defined` typically mean?
A: You referenced a name that was never assigned in scope — often a typo, a variable defined in another function, or used before assignment.

Q: What does `AttributeError: 'NoneType' object has no attribute 'strip'` usually reveal?
A: A value you expected to be a string is actually `None` — frequently a function returned `None` or a `.get()` missed a key.

Q: What is an HTTP status code, and what does the 2xx range mean?
A: It's a three-digit number summarizing a request's outcome; 2xx (e.g. `200 OK`, `201 Created`) means the request succeeded.

Q: What do the 4xx and 5xx status-code ranges mean, with one example each?
A: 4xx is a client error (e.g. `404 Not Found`, `401 Unauthorized`), 5xx is a server error (e.g. `500 Internal Server Error`, `503 Service Unavailable`).

Q: What does `429 Too Many Requests` mean and how should a client respond?
A: You've been rate-limited; back off and retry after a delay, ideally honoring the `Retry-After` header rather than hammering the endpoint.

Q: Why must you always set a timeout on an HTTP request?
A: Without a timeout a hung server can block your program indefinitely; a timeout makes the call fail fast so your loader can retry or report cleanly.

Q: After `resp = requests.get(url)`, how do you turn a 4xx/5xx into an exception?
A: Call `resp.raise_for_status()`, which raises an `HTTPError` for error status codes instead of letting you silently process a failed response.

Q: How do you parse a JSON response body into Python objects?
A: Call `resp.json()` (or `json.loads(resp.text)`), which converts the JSON into dicts, lists, strings, numbers, and booleans.

Q: What is the difference between `json.loads` and `json.load`?
A: `json.loads` parses a JSON string in memory, while `json.load` reads and parses from a file object; the `s` stands for "string".

Q: How does JSON map onto Python types when you parse it?
A: Object -> `dict`, array -> `list`, string -> `str`, number -> `int`/`float`, `true`/`false` -> `bool`, `null` -> `None`.

Q: Why should you check `resp.status_code` (or call `raise_for_status`) before calling `resp.json()`?
A: An error response often returns HTML or an error JSON, so parsing it as your expected payload yields a confusing `KeyError` or decode error downstream.

Q: What does `json.dumps(data, indent=2)` produce versus `json.dumps(data)`?
A: `indent=2` pretty-prints with newlines and indentation for human reading, while the plain form produces compact single-line JSON.

Q: When writing JSON for fund names with accents, what argument keeps them readable?
A: Pass `ensure_ascii=False` to `json.dumps` so non-ASCII characters are written as themselves rather than `\uXXXX` escapes.

Q: A NAV feed sometimes returns `200` with an empty body — why is `resp.json()` risky there?
A: An empty body isn't valid JSON, so `resp.json()` raises a `JSONDecodeError`; guard by checking the body or content length before parsing.

Q: How would you safely retry a transient `503` from a pricing API?
A: Wrap the call in a loop with a bounded number of attempts and an increasing (exponential) backoff delay, only retrying idempotent requests and surfacing a clear error if all attempts fail.

Q: What is `argparse` and why use it instead of reading `sys.argv` directly?
A: `argparse` is the standard library's command-line parser; it gives you typed flags, defaults, validation, usage errors, and `--help` for free instead of hand-parsing `sys.argv`.

Q: How do you add a `--currency` flag to an `ArgumentParser`?
A: `parser.add_argument("--currency")`, optionally with `help=...`, `default=...`, and `choices=[...]` to constrain and document it.

Q: How do you make argparse treat a flag as a boolean switch like `--verbose`?
A: Use `action="store_true"`, so the flag's value is `True` when present and `False` (the default) when absent.

Q: Where does `--help` text come from, and why does the plan require good help?
A: It's generated from the parser's `description` and each argument's `help=`; good help lets someone run `fundcli` correctly without reading the source.

Q: What is an exit code, and what do `0` and non-zero mean to a calling shell or pipeline?
A: It's the integer a process returns on exit; `0` means success and any non-zero value means failure, which scripts and CI use to decide whether to continue.

Q: How do you set a specific exit code from your Python CLI?
A: Call `sys.exit(2)` (or return that code from `main` to a caller that exits), or `raise SystemExit("message")` to print and exit non-zero.

Q: How do you restrict `--currency` to a known set of values in argparse?
A: Pass `choices=["EUR", "USD", "GBP"]`; argparse rejects anything else with a usage error and exit code `2`.

Q: Why print error messages to stderr rather than stdout in a CLI?
A: stdout carries the program's real output (e.g. the JSON summary) for piping, while stderr carries diagnostics, so errors don't pollute the data stream.

Q: What is ruff and what does it do?
A: ruff is a fast Python linter and formatter (from Astral) that flags style issues, likely bugs, and unused imports, and can auto-format code.

Q: What does `ruff check .` do versus `ruff format .`?
A: `ruff check .` lints and reports (and with `--fix` auto-corrects) rule violations, while `ruff format .` reformats code to a consistent style.

Q: Why wire ruff into the editor to run on every save from day one?
A: Continuous linting catches mistakes and enforces consistent style immediately, so issues never accumulate and reviews focus on logic, not formatting.

Q: A teammate runs `python fundcli.py` and gets `ModuleNotFoundError` for a dependency you added — what's the likely fix?
A: They ran bare `python` outside the project environment; running `uv run fundcli.py` (after `uv sync`) uses the locked environment where the dependency is installed.

Q: Your loader prints `currency` counts as all `1` even with duplicates — first thing to check?
A: Check whether you're counting unique values (e.g. accidentally building a set or using a dict keyed by currency overwriting counts) instead of incrementing a `Counter`.

Q: `uv run fundcli.py funds.csv` exits 0 but writes no JSON for a malformed file — what's wrong with the design?
A: Errors are being swallowed instead of raised; a malformed row should raise with its row number and the process should exit non-zero, per the loader's "fail loudly" spec.
